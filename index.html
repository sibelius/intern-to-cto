My Story

I started my career as an intern developer right out of college. I was eager to learn and grow in my role, and I was fortunate enough to work with a team that was supportive and invested in my development.

Over the years, I worked my way up through the ranks, taking on more responsibility and challenging projects. I always looked for opportunities to learn and improve, whether it was through additional training, attending conferences, or collaborating with colleagues.

Eventually, I was promoted to a leadership role as a CTO. It was a big step up, but I felt prepared for the challenge thanks to the foundation I had built as an intern developer.

Lessons Learned

Here are some key lessons I learned along the way:

  1. Be a lifelong learner - the tech industry is constantly evolving, so it's important to stay up-to-date with the latest trends and technologies.

  2. Build relationships - networking and building relationships with colleagues, mentors, and other industry professionals can open doors and provide valuable support and guidance.

  3. Be proactive - don't wait for opportunities to come to you, seek them out and take the initiative to learn and grow.

  4. Develop leadership skills - as you progress in your career, you'll need to develop skills like communication, delegation, and strategic thinking to succeed in leadership roles.

Conclusion

In conclusion, going from an intern developer to becoming a CTO is a challenging but rewarding journey. By focusing on continuous learning, building relationships, taking initiative, and developing leadership skills, you can set yourself up for success in any role you choose.

What are types?

  • Just sets
  • Set theory

Primitive types

  • Built-in types
  • number
  • string
  • boolean
  • symbol
  • bigint
  • undefined
  • null

Literal types

  • The exact types
  • "apple"
  • 1
  • true

Data structures

  • Objects: { a: string }
  • Records: { [key: string]: string }
  • Tuples: [string, number]
  • Array: string[]

Union types

  • Join types together
  • a | b

Intersection types

  • What belongs to both types
  • a & b

Special types

  • unknown
  • never
  • any

Unknown type

  • Contains every type
  • a | unknown = unknown
  • a & unknown = a

Never type

  • Empty set
  • string & number
  • a | never = a
  • a & never = never
  • const crash = (): never => process.exit(1);

Any type

  • Doesn't fit in the set theory
  • Both supertype and subtype of every other type
  • Bad practice
  • a & any = any
  • a | any = any

Conditional types

  • extends
  • "Hello" extends string ? true : false

Infer types

  • { a: string } extends { a: infer V } ? V : never

Template literal types

  • type T = `$0 (${number})`

Examples

Naturals

type Zero = 'Zero';
type Suc<N extends Nat> = `Suc<${N}>`;
type Nat = Zero | Suc<any>;
type Add<X extends Nat, Y extends Nat> = Y extends Zero
  ? X
  : Y extends Suc<infer YP>
  ? Add<Suc<X>, YP>
  : Zero;
type a1 = Add<Suc<Zero>, Suc<Zero>>;
//   ^? type a1 = "Suc<Suc<Zero>>" -> 3
type Sub<X extends Nat, Y extends Nat> = X extends Suc<infer XP>
  ? Y extends Suc<infer YP>
    ? Sub<XP, YP>
    : X
  : Zero;
type s1 = Sub<Suc<Suc<Zero>>, Suc<Zero>>;
//   ^? type s1 = "Suc<Zero>" -> 1
type Mult<X extends Nat, Y extends Nat> = Y extends Suc<infer YP>
  ? Add<X, Mult<X, YP>>
  : Zero;
type m1 = Mult<Suc<Suc<Zero>>, Suc<Suc<Zero>>>;
//   ^? type m1 = "Suc<Suc<Suc<Suc<Zero>>>>" -> 4
type _Div<
  X extends Nat,
  Y extends Nat,
  Q extends Nat,
  R extends Nat
> = X extends Suc<infer XP>
  ? R extends Suc<infer RP>
    ? _Div<XP, Y, Q, RP>
    : _Div<XP, Y, Suc<Q>, Y>
  : Q;
type Div<X extends Nat, Y extends Nat> = Y extends Suc<infer YP>
  ? _Div<X, YP, Zero, YP>
  : Zero;
type d1 = Div<Suc<Suc<Suc<Suc<Suc<Suc<Zero>>>>>>, Suc<Suc<Zero>>>;
//   ^? type d1 = "Suc<Suc<Suc<Zero>>>" -> 3

T function options

type TemplateOptions<S extends string> =
  S extends `${string}{${infer K}}${infer R}` ? TemplateOptions<R> | K : never;
type t1 = TemplateOptions<'Translation {count} {anotherKey}'>;
//   ^? type t1 = "count" | "anotherKey"
